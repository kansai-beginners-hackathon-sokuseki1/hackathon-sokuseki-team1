# Project Board Rules

## この文書で一番大事なこと
- 進捗の正本は GitHub Project に置く。
- すべての作業は Issue と紐づける。
- 状態は `Todo / In Progress / Review / Done` の4つで揃える。

## 列の意味
- `Todo`: まだ着手していない
- `In Progress`: 誰かが作業中
- `Review`: PR 作成済み、または確認待ち
- `Done`: 完了

## 運用ルール
- タスクを作ったら `Todo` に入れる
- 着手したら `In Progress` に移す
- PR を出したら `Review` に移す
- `main` に反映されたら `Done` に移す

## 最初に作る Issue の例
- 何を作るかの候補整理
- 対象ユーザー整理
- MVP の決定
- 技術選定
- 役割分担
- 発表内容の決定
