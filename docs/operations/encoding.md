# Encoding Policy

## 基本方針
- このリポジトリのテキストファイルは UTF-8 で保存する。
- 文字化けした状態のままコミットしない。
- 改行コードは原則 `LF` を使う。

## 対象
- Markdown
- JavaScript / JSX / CSS
- JSON / YAML
- 設定ファイル全般

## 運用ルール
- 新規作成ファイルは UTF-8 で保存する。
- 既存ファイルを編集する際に文字化けを見つけたら、その変更の中で直す。
- エディタやツールの設定で UTF-8 を既定値にする。
- 文字コードが不明なファイルを扱う場合は、編集前に UTF-8 で正しく読める状態を確認する。
