# AI Agent Rules

## この文書で一番大事なこと
- AI に渡す作業も Issue や明文化タスクに紐づける。
- AI の変更は個別ブランチ上ではレビュー不要とし、`main` に入れる時だけ人がレビューする。
- AI にも秘密情報の直書き禁止、UTF-8 方針、ブランチ運用を適用する。

## AI へ依頼する時に書くこと
- 目的
- 完了条件
- 制約
- 変更してよい範囲
- 変更してはいけない範囲

## 守るルール
- `main` へ直接変更を入れさせない
- API キー、トークン、エンドポイントを直書きさせない
- テスト追加または確認手順の記載を求める
- 意思決定に関わる出力は repo の文書へ転記する

## レビュー
- AI 生成物は `main` へマージする PR の段階で確認する
- `動くか`
- `安全か`
- `意図どおりか`
- `不要な変更がないか`
